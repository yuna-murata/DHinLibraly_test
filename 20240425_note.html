TEIとは
  • Text Encoding Initiative（TEIガイドライン）
    • 人文学分野を主としたデジタル資料の構造的記述のためのガイドライン
    • 標準規格ではなく、ガイドライン

      →述語体系が学問分野によって異なるという人文学の性質を考慮し、実際の用法は利用者にゆだねる形をとる。よって、同じテキスト要素のマークアップにも様々な方法があり得る。

  • 歴史・背景
    • 1980年代　電子テキストのタグ付けの取り組み
      • →プロジェクトによりタグの付け方はバラバラ、共通化の必要性
    • 1987年　ポキプシー原則の策定
      • 人文学研究におけるデータ交換のための標準的な形式を提供すること等を取り決めた
      • 当初はXSLTベース→XMLベースに
    • TEI ガイドラインの策定
    • TEI 協会（Consortium）を設置
    • TEI P5（現在の最新バージョン）

    • 写本等、従来のMARCでは表現が難しいメタデータの記述が柔軟にできる

XMLの概要
  • テキストの意味や構造を記述（マークアップ）するための言語
  • 「タグ」によってテキストの意味や構造を表現する
  • エレメント（要素）とアトリビュート（属性）
  • XMLの制約：タグは入れ子構造にできるが、オーバーラップはできない
  • TEIガイドラインは、テキストのどのような情報をどのようなタグや属性を使ってマークアップするかを定めたもの。
  • TEI/XML：TEIガイドラインに沿って記述されたXML文書のこと
TEI/XMLファイルの構成
  • XML宣言：この文書がXML文書であることを宣言
  • 名前空間：この文書がTEIの名前空間におけるタグを使用していることを表現
  • TEIヘッダ：メタデータを記述
  • text：データ本体を記述
<?xml version="1.0" encoding="UTF-8"?>
<TEI xmlns="http://www.tei-c.org/ns/1.0">
<teiHeader>
メタデータ
</teiHeader>
<text>
コンテンツ
</text>
</TEI>

マークアップする要素の検討
  • メタデータ
    • メタデータはteiHeader要素内に記述する
    • 各タグの使用方法の詳細はTEIガイドライン第2章（日本語訳）及び参考文献2のChap4-3を参照
    • teiヘッダは主に以下の5つの要素からなる
      • filedesc★：コンピュータファイル自体の完全な書誌学的記述を含み、そこからテキストの利用者が適切な書誌学的引用情報を導き出すことができるもの。
      • sourceDesc★：底本に関する情報を記述する
      • encodingDesc★：テキストの正規化の方法や、符号化する者がソースの曖昧さをどのように解決したか、どのレベルの符号化や解析が適用されたかなどを記述する
      • ProfileDesc：そのテキストの主題、それが作成された状況、それを作成した人物、またはそれの作成に参加した人物など、テキストに関する分類情報と文脈情報を含む。
      • xenoData：非TEIメタデータを配置できる。
      • RevisionDesc：改訂履歴、バージョン等に関する情報

  • 校異情報
    • TEIヘッダの<sourceDesc>タグ内に<listwit>で諸本の情報を記述
    • <app>：校異情報がある箇所全体を囲む
    • <lem>：本文（底本）の表現
    • <rdg>：異文の表現
    • ＠wit：その表現が現れる資料のIDを参照

    ①まずは諸本（今回は京都本と国会本）の表現を比較する

    →赤字が異同のある箇所

    #京都本の記述
あはさぬきに平家をそむき源氏をまちけるもの共
かしこのほらこゝのたによりはせきたつてくははる源氏

#国会本の記述
あふぎのまと
あわさぬきに平家をそむき源氏をまちけるものどもかしこのほら
こゝのたによりはせきたつてくわゝる源氏のせいほともなく三百よき

    ②これをTEIで表現すると次のようになる。

    <teiHeader>
	<sourceDesc>
	<listWit>
	<witness xml:id="京都本">百二十句本　京都本</witness>
	<witness xml:id="国会本">百二十句本　国会本</witness>
	</listWit>
	</sourceDesc>
</teiHeader>

  • デジタル画像のリンク
    • 参考：TEI-EAJの解説
    • 前準備
      • IIIFマニフェストURLの作成
    • マークアップ
    1. facsimile 要素内に surfaceGrp 要素を追加する。その facs 属性に IIIF Manifest の URI を指定する。
    <TEI xmlns="http://www.tei-c.org/ns/1.0">
　　　<teiHeader>...</teiHeader>
　　　<facsimile>
　　　　　　<surfaceGrp
　　　　　　　　　facs="https://iiif.dl.itc.u-tokyo.ac.jp/repo/iiif/102a847a-ea96-464a-84ba-cb714696bfbd/manifest">
　　　　　　...
　　　　　　</surfaceGrp>
　　　</facsimile>
　　　<text>...<text>
</TEI>
    1. surfaceGrp 要素内に surface 要素、 graphic 要素を追加する。その url 属性に画像の URL を指定し、n 属性にカンバス ID を指定する。
    <surfaceGrp ...>
　　　<surface>
　　　　　　<graphic url="https://iiif.dl.itc.u-tokyo.ac.jp/iiif/koshu/images/koshu.tif/full/8000,/0/default.jpg"
　　　　　　　　　n="https://iiif.dl.itc.u-tokyo.ac.jp/repo/iiif/102a847a-ea96-464a-84ba-cb714696bfbd/canvas/p1"/>
　　　</surface>
　　　<surface> ... </surface>
</surfaceGrp>
    1. surface 要素内に zone 要素を追加する。その xml:id 属性に ID を与え、矩形の左下（lrx 属性と lry 属性）と右上（ulx 属性と uly 属性）の点を指定する。
    <surface>
　　　<graphic  ... />
　　　<zone lrx="7613" lry="237" ulx="7568" uly="0" xml:id="zone_l0001_l_0001"/>
　　　<zone ... />
</surface>

    4.特定の要素内に facs 属性を与え、テキストと画像を関連づける。

    <span facs="#zone_l0001_l_0001">　　 甲州法度之次第</span>
